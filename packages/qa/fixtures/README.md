# QA Fixtures

This directory stores reusable, non-sensitive QA assets. It does not store run output.

Appropriate fixtures can include maintained workspace templates, small database seeds, provider-binary shims, or static
payloads that are useful across multiple QA tasks.

Per-run seed payloads, screenshots, logs, reports, bug artifacts, database dumps, downloaded binaries, generated
workspaces, and temporary source material belong in the temporary run directory, not in this package.
