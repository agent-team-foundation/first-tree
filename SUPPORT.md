# Support

Use the right channel for the kind of help you need:

- Usage questions, onboarding confusion, or proposal interpretation:
  open a GitHub Discussion if available, or file an issue with the `question`
  framing.
- Bugs in the CLI, docs, or runtime behavior:
  open a GitHub issue with reproduction steps, expected behavior, and actual behavior.
- Security-sensitive reports:
  follow [SECURITY.md](./SECURITY.md) instead of opening a detailed public issue.

## What helps maintainers most

When reporting a problem, include:

- the command path you ran
- whether you are on the published package or a source checkout
- the relevant version or commit
- the exact output or failure mode
- which surface the issue is about (`tree`, `github scan`, agent / chat
  collaboration, web console, or docs)
