# Proposals

Temporary files and process documents.

---

## Purpose

A place for draft proposals, design explorations, and other in-progress documents. These files have a limited lifespan — once a proposal is adopted, its conclusions should be written into the corresponding domain node (NODE.md), not left here.

---

## Agent Access Rules

**Agents must not read files in this directory as knowledge or context.**

Contents here are temporary, potentially outdated, and not finalized. When building context or making decisions, agents should rely on each domain's NODE.md as the source of truth.

The only exception: when a human explicitly asks an agent to read a specific file under proposals.

---

## Naming Convention

`<topic>.<date>.md`, e.g. `agent-message-system.20260319.md`. The date helps assess timeliness.
